## Cloudflare Workers

The API runs on Cloudflare Workers and is deployed via Wrangler.

### Prerequisites

- Node.js v18+
- pnpm v8+
- Cloudflare account with Workers enabled

### Configuration

The Worker configuration lives in `api/wrangler.toml`:

```toml
name = "ollama-models-api"
main = "src/index.ts"
compatibility_date = "2024-11-01"
# account_id is provided via CLOUDFLARE_API_TOKEN or --account-id flag at deploy time.

[vars]
OLLAMA_BASE = "https://ollama.com"
OLLAMA_USER_AGENT = "ollama-models-api/0.1 (+https://github.com/devcomfort/ollama-models)"
OLLAMA_ACCEPT = "text/html,application/xhtml+xml"
OLLAMA_ACCEPT_LANGUAGE = "en-US,en;q=0.9"
```

### Deploy

```bash
pnpm deploy
```

This runs `wrangler deploy` from the `api/` directory. Wrangler transpiles TypeScript at deploy time — no separate build step is needed.

## Observability

Scraper failures are logged as structured JSON via `console.error()` and surfaced in Cloudflare Workers Logs. Runtime errors are also sent as email alerts via a Tail Worker using Cloudflare Email Service.

### Runtime Error Alerts (Cloudflare Email Service)

A Tail Worker (`workers/alerts/`) receives real-time execution events from the API Worker and sends email alerts for any non-`ok` outcome.

**Setup:**

1. **Email Routing — verify your email:**
   - Dashboard → Email Routing → Destination addresses → Add address
   - Enter your email → click the confirmation link

2. **Email Sending — onboard domain:**
   - Dashboard → Compute → Email Service → Email Sending → Onboard Domain
   - Select `devcomfort.me` (Cloudflare auto-adds SPF, DKIM, DMARC records)

3. **Set the recipient secret:**
```bash
cd workers/alerts
wrangler secret put ALERT_EMAIL_TO
# Enter your email address
```

4. **Deploy the Tail Worker:**
```bash
wrangler deploy
```

### CI/CD Failures

CI failures (deploy, E2E, health check) are visible in the GitHub Actions UI. No external notification channel is configured.

## Environment Variables

| Variable | Purpose | Set via |
|----------|---------|--------|
| `OLLAMA_BASE` | Base URL for ollama.com scraping | `wrangler.toml` `[vars]` |
| `OLLAMA_USER_AGENT` | User-Agent header for upstream requests | `wrangler.toml` `[vars]` |
| `OLLAMA_ACCEPT` | Accept header for upstream requests | `wrangler.toml` `[vars]` |
| `OLLAMA_ACCEPT_LANGUAGE` | Accept-Language header for upstream requests | `wrangler.toml` `[vars]` |


## CI/CD

### Pull Request Workflow

```yaml
# ci.yml — runs on every push and PR
- API: pnpm test:api (89 unit + integration tests)
- TS client: pnpm test:ts (28 tests) + smoke test
- Python: pytest (31 unit) + 8 integration tests
```

### Deploy Workflow

```yaml
# deploy.yml — triggered on main push (api/, packages/, docs/, scripts/)
- deploy-api     → Deploy to Cloudflare Workers
- health-check   → Probe /health, /search, /model
- e2e            → Full E2E suite (23 tests)
- deploy-docs    → Build Astro → Deploy to Cloudflare Pages
- deploy-ts-client → Test → Build → Smoke test → Publish to npm
- deploy-py-client → Handled by publish-pypi.yml on py-v* tag
```

CI failures are visible in the GitHub Actions UI. Documentation-only commits skip deployment.
