## Cloudflare Workers

API는 Cloudflare Workers에서 실행되며 Wrangler를 통해 배포됩니다.

### 사전 요구사항

- Node.js v18+
- pnpm v8+
- Workers가 활성화된 Cloudflare 계정

### 설정

Worker 설정은 `api/wrangler.toml`에 있습니다:

```toml
name = "ollama-models-api"
main = "src/index.ts"
compatibility_date = "2024-11-01"
# account_id는 배포 시 CLOUDFLARE_API_TOKEN 또는 --account-id 플래그로 제공됩니다.

[vars]
OLLAMA_BASE = "https://ollama.com"
OLLAMA_USER_AGENT = "ollama-models-api/0.1 (+https://github.com/devcomfort/ollama-models)"
OLLAMA_ACCEPT = "text/html,application/xhtml+xml"
OLLAMA_ACCEPT_LANGUAGE = "en-US,en;q=0.9"
```

### 배포

```bash
pnpm deploy
```

`api/` 디렉토리에서 `wrangler deploy`를 실행합니다. Wrangler는 배포 시 TypeScript를 트랜스파일하므로 별도의 빌드 단계가 필요하지 않습니다.

## 관측성

스크래퍼 실패는 `console.error()`를 통해 구조화된 JSON으로 로깅되며 Cloudflare Workers Logs에서 확인할 수 있습니다. 런타임 에러는 Tail Worker가 Cloudflare Email Service를 통해 이메일로도 전송합니다.

### 런타임 에러 알림 (Cloudflare Email Service)

Tail Worker(`workers/alerts/`)가 API Worker의 실행 이벤트를 실시간으로 수신하여 `ok`가 아닌 결과에 대해 이메일 알림을 보냅니다.

**설정:**

1. **Email Routing — 이메일 주소 검증:**
   - 대시보드 → Email Routing → Destination addresses → Add address
   - 이메일 입력 → 확인 링크 클릭

2. **Email Sending — 도메인 온보딩:**
   - 대시보드 → Compute → Email Service → Email Sending → Onboard Domain
   - `devcomfort.me` 선택 (Cloudflare가 SPF, DKIM, DMARC 레코드 자동 추가)

3. **수신자 시크릿 설정:**
```bash
cd workers/alerts
wrangler secret put ALERT_EMAIL_TO
# 이메일 주소 입력
```

4. **Tail Worker 배포:**
```bash
wrangler deploy
```

### CI/CD 실패

CI 실패(배포, E2E, 헬스 체크)는 GitHub Actions UI에서 확인할 수 있습니다. 별도의 외부 알림 채널은 설정되어 있지 않습니다.

## 환경 변수

| 변수 | 목적 | 설정 방법 |
|------|------|-----------|
| `OLLAMA_BASE` | ollama.com 스크래핑용 Base URL | `wrangler.toml` `[vars]` |
| `OLLAMA_USER_AGENT` | 업스트림 요청용 User-Agent 헤더 | `wrangler.toml` `[vars]` |
| `OLLAMA_ACCEPT` | 업스트림 요청용 Accept 헤더 | `wrangler.toml` `[vars]` |
| `OLLAMA_ACCEPT_LANGUAGE` | 업스트림 요청용 Accept-Language 헤더 | `wrangler.toml` `[vars]` |


## CI/CD

### Pull Request 워크플로우

```yaml
# ci.yml — 모든 push와 PR에서 실행
- API: pnpm test:api (89개 유닛 + 통합 테스트)
- TS 클라이언트: pnpm test:ts (28개 테스트) + smoke test
- Python: pytest (31개 유닛) + 8개 통합 테스트
```

### 배포 워크플로우

```yaml
# deploy.yml — main push 시 트리거 (api/, packages/, docs/, scripts/)
- deploy-api        → Cloudflare Workers에 배포
- health-check      → /health, /search, /model 프로브
- e2e               → 전체 E2E 스위트 (23개 테스트)
- deploy-docs       → Astro 빌드 → Cloudflare Pages에 배포
- deploy-ts-client  → 테스트 → 빌드 → Smoke test → npm에 배포
- deploy-py-client  → publish-pypi.yml이 py-v* 태그에서 처리
```

CI 실패는 GitHub Actions UI에서 확인할 수 있습니다. 문서 전용 커밋은 배포를 건너뜁니다.

### 로컬 개발

| 명령어 | 설명 |
|---------|-------------|
| `pnpm dev` | API 로컬 개발 서버 시작 |
| `pnpm docs:dev` | 문서 사이트 로컬 실행 |
| `pnpm build:docs` | 문서 사이트 빌드 |
| `pnpm type-check` | 전체 TypeScript 타입 체크 |
