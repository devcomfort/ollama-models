## Cloudflare Workers

API는 Cloudflare Workers에서 실행되며 Wrangler를 통해 배포됩니다.

### 사전 요구사항

- Node.js v18+
- pnpm v8+
- Workers가 활성화된 Cloudflare 계정

### 설정

Worker 설정은 `api/wrangler.toml`에 있습니다:

```toml
name = "ollama-models-api"
main = "src/index.ts"
compatibility_date = "2024-11-01"

account_id = "your-account-id"

[triggers]
crons = ["0 * * * *"]

[vars]
OLLAMA_BASE = "https://ollama.com"
OLLAMA_USER_AGENT = "ollama-models-api/0.1 (+https://github.com/devcomfort/ollama-models)"
OLLAMA_ACCEPT = "text/html,application/xhtml+xml"
OLLAMA_ACCEPT_LANGUAGE = "en-US,en;q=0.9"
```

### 배포

```bash
pnpm deploy
```

`api/` 디렉토리에서 `wrangler deploy`를 실행합니다. Wrangler는 배포 시 TypeScript를 트랜스파일하므로 별도의 빌드 단계가 필요하지 않습니다.

## 시크릿

알림용 웹훅 URL은 Cloudflare Workers 시크릿으로 저장됩니다:

```bash
# 알림 웹훅 URL 설정 (Slack, Discord 또는 이메일)
wrangler secret put ALERT_WEBHOOK_URL --config api/wrangler.toml
```

시크릿은 암호화되며 `wrangler.toml`이나 버전 관리에 나타나지 않습니다.

## Cron 트리거

Worker는 매시간 예약된 헬스 체크를 실행합니다:

```toml
[triggers]
crons = ["0 * * * *"]
```

각 cron 틱마다 `runHealthCheck()`가 두 스크래퍼를 프로브합니다. 프로브가 실패하면 설정된 웹훅 URL로 알림이 전송됩니다.

## 환경 변수

| 변수 | 목적 | 기본값 |
|----------|---------|---------|
| `OLLAMA_BASE` | ollama.com 스크래핑용 Base URL | `https://ollama.com` |
| `OLLAMA_USER_AGENT` | 업스트림 요청용 User-Agent 헤더 | `ollama-models-api/0.1 (+https://github.com/devcomfort/ollama-models)` |
| `OLLAMA_ACCEPT` | 업스트림 요청용 Accept 헤더 | `text/html,application/xhtml+xml` |
| `OLLAMA_ACCEPT_LANGUAGE` | 업스트림 요청용 Accept-Language 헤더 | `en-US,en;q=0.9` |
| `ALERT_WEBHOOK_URL` | 실패 알림용 웹훅 URL (시크릿) | 설정 안 함 |

## CI/CD

### Pull Request 워크플로우

```yaml
# ci.yml
- 의존성 설치 (pnpm install)
- 유닛 테스트 실행 (pnpm test)
- 통합 테스트 실행 (pnpm test:api, pnpm test:ts, pnpm test:py)
```

### 배포 워크플로우

```yaml
# deploy.yml (main push 시 트리거)
- API 배포 (pnpm deploy)
- 배포된 인스턴스에 대해 헬스 체크 실행
- E2E 테스트 실행 (./scripts/e2e.sh)
- 패키지 퍼블리시 (npm, PyPI)
```

배포 워크플로우는 코드 파일이 변경되었을 때만 트리거됩니다 — 문서 전용 커밋은 배포를 건너뜁니다.
